About this project

This project demonstrates my expertise in DITA (Darwin Information Typing Architecture) and Oxygen XML Editor, with a focus on structured, reusable, and topic-based documentation. I am actively deepening this expertise through hands-on project work that showcases my ability to produce well-organised, consistent documentation using DITA's structured framework.

Core principle: In DITA, every topic is self-contained and answers exactly one question. Content reuse via conrefs means you write once and publish everywhere — without copy-paste drift. This discipline is what separates scalable documentation from one-off deliverables.

What DITA structured authoring involves

  • Topic types — Task (how to do it), Concept (what it is), Reference (look-up data). Each type has a strict schema that enforces discipline in content creation.
  • Content reuse with conrefs — A single source element referenced across multiple topics. When the source changes, every reference updates automatically.
  • DITA maps — The navigation layer that assembles individual topics into a coherent output, independently of topic order or content.
  • Multi-channel publishing — The same topic set can produce HTML5 web help, PDF, and mobile outputs from a single source without reformatting.
  • Validation in Oxygen XML — Real-time schema validation ensures every topic conforms to DITA 1.3 standards before publishing.

Tools & standards

  • Oxygen XML Editor — Industry-standard DITA authoring environment; XML editing, DITA validation, topic-based authoring, and transformation scenarios
  • DITA 1.3 — The current OASIS standard for structured technical documentation
  • DITA Open Toolkit — Transformation engine for generating HTML5, PDF, and other outputs

Skills demonstrated

  • Structured authoring — Creating content that is modular, reusable, and topic-typed from the ground up
  • Information architecture — Designing content maps that scale without becoming unmaintainable
  • XML proficiency — Working directly in XML with schema awareness, not just through a visual editor
  • Single-source publishing — One content set producing multiple output formats without manual reformatting

Why DITA matters

DITA is the standard for enterprise technical documentation in aerospace, medical devices, manufacturing, and software. Organisations that adopt DITA reduce translation costs (you only translate what changed), accelerate publishing cycles, and maintain consistency across documentation sets of any size. As a DITA practitioner, I can work within existing DITA environments or design new ones from scratch.
