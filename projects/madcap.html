About this project

This project is a technical documentation and web styling showcase: a well-organised digital guide on Nicholas Sparks' books, movies, and foundation, built with MadCap Flare. The subject matter is intentionally non-technical — the focus is entirely on demonstrating MadCap Flare expertise, custom CSS, and single-source publishing discipline.

The project produces HTML5 web help and PDF from the same source, with breadcrumbs, sidebars, and responsive design implemented through Flare's built-in features and custom stylesheet overrides.

Single-source discipline: The same content set produces both the HTML5 web output and the PDF — with zero manual reformatting. Conditional tags control what appears in which output. Stylesheets control how it looks in each format independently.

Technologies & tools used

  • MadCap Flare — Topic-based authoring with modular content, breadcrumbs, sidebars, responsive design. Leveraged Flare's full feature set: conditional tags, TOC design, cross-references, and transformation targets.
  • Custom CSS — Adjusted fonts, colour schemes, layouts, and button styles. Used pseudo-elements, flexbox layouts, and custom colour schemes to achieve a polished, consistent aesthetic.
  • GitHub — Full version control and project management throughout the publication process.

Key technical skills demonstrated

  • Content structuring — Topics and subtopics organised using best practices in technical documentation: modular, searchable, and logically structured
  • Single-source publishing — HTML5 web help and PDF from one source; conditional tags control audience-specific content
  • Responsive design — Layouts, images, and fonts adapt to screen size; tested across devices
  • Advanced CSS styling — Custom pseudo-elements, flexbox, consistent colour system across all pages
  • Repository management — GitHub version control, change tracking, and open publishing

What this demonstrates for professional TW

MadCap Flare is the industry-standard help authoring tool for mid-to-large software and hardware documentation teams. Proficiency in Flare — including CSS customisation, conditional publishing, and responsive output — is a core requirement for senior technical writer roles. This project demonstrates end-to-end fluency: from content structure to styled, published output.
