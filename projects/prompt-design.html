About this project

This is a comprehensive study on prompt design strategies for Generative AI tools, developed as the capstone of my diploma project in Technical Communication. The guide offers detailed insights and practical examples on designing effective prompts to maximise the accuracy and relevance of responses from generative AI models.

The project compares how three leading AI models — ChatGPT, Gemini, and Copilot — respond to identical prompts, showcasing how small differences in prompt structure, style, and specificity produce dramatically different outputs.

Key insight: The same question phrased differently can yield outputs that differ in accuracy, depth, and format by an order of magnitude. This project documents exactly why — and how to engineer prompts that consistently produce what you need.

What it covers

  • Role assignment — how defining the model's persona changes output quality
  • Few-shot examples — showing the model what "good" looks like before asking
  • Chain-of-thought prompting — getting step-by-step reasoning for complex tasks
  • Output format control — specifying structure, length, and formatting precisely
  • Summarisation prompts — extracting key information without losing nuance
  • Creative generation — balancing creativity with consistency and brand voice
  • Problem-solving prompts — structured reasoning for technical documentation challenges

Technologies & tools used

  • Markdown, HTML, XML — content structure and formatting, clean and logically organised
  • CSS customisation with Just The Docs — modified theme for improved readability, colour-coded prompt examples, and user-centred layout
  • ChatGPT, Gemini, Copilot — tested with identical prompts to demonstrate response differences
  • Snagit & GIMP — annotated screenshots and comparison tables showing model responses side-by-side
  • GitHub Pages — version control and public documentation publishing

Key skills demonstrated

  • Prompt engineering — practical understanding of how prompt structure affects model output, across multiple tools and task types
  • Multimodal documentation — Markdown, XML, and HTML layered to create navigable, structured content
  • CSS styling & customisation — Just The Docs theme modified for colour-coded sections, clean navigation, professional finish
  • Visual annotation — Snagit and GIMP used to create side-by-side AI output comparisons
  • Docs-as-code publishing — full version control and GitHub Pages deployment workflow

Why this matters for TW

Technical writers who understand prompt engineering have a measurable advantage: they can use AI tools to accelerate drafting, review, localisation review, and quality checking — without losing authorial control over accuracy and tone. This project documents that understanding at a level that goes beyond "tips and tricks" into principled, repeatable methodology.
